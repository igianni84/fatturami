## Codebase Patterns
- Stack: Next.js 16 (App Router), TypeScript, Tailwind CSS 4, Prisma 6 ORM, PostgreSQL
- Use Zod for form and API validation
- UI language: Italian
- Document generation languages: ES, IT, EN
- Docker Compose for deployment (app + PostgreSQL)
- File storage: local filesystem (Docker volume)
- Single-user system (no multi-tenancy)
- Monetary amounts stored as Decimal in Prisma
- Status fields as Prisma enums
- Soft delete pattern: deletedAt timestamp field
- Progressive numbering: PREFIX-YYYY-NNN format
- VAT regime auto-detection based on client country
- Prisma client import: `import { PrismaClient } from "@prisma/client"` (Prisma 6 style, enums also imported from there)
- Seed script uses `tsx` runner: configured via `prisma.seed` field in package.json
- Seed idempotency pattern: use deterministic IDs with `upsert` (e.g., `seed-{type}-{country}`)
- Prisma singleton pattern in `src/lib/prisma.ts`
- Tailwind 4 uses `@import "tailwindcss"` in globals.css (no tailwind.config.js needed)
- PostCSS uses `@tailwindcss/postcss` plugin
- Next.js output mode: "standalone" (for Docker deployment)
- ESLint uses flat config (eslint.config.mjs) with next/core-web-vitals
- Migration generation without DB: `npx prisma migrate diff --from-empty --to-schema-datamodel prisma/schema.prisma --script`
- Pages with DB queries need `export const dynamic = "force-dynamic"` to avoid build-time pre-rendering errors
- Server action pattern: Zod schema + safeParse validation + return `{ success, errors?, message? }`
- Form pages: server component loads data, passes to client component form via props
- Line items pattern: description, quantity, unitPrice, taxRateId (except PurchaseInvoiceLine which uses amount)
- All parent-child relations use onDelete: Cascade for line items
- Sidebar navigation component at `src/components/Sidebar.tsx` (client component with usePathname for active highlighting)
- Layout structure: flex h-screen with fixed sidebar (w-64) and scrollable main content area
- ESLint has a circular reference issue with FlatCompat + next/core-web-vitals (pre-existing, not blocking)
- Prisma singleton import: `import { prisma } from "@/lib/prisma"` (named export, NOT default)
- DataTable component at `src/components/DataTable.tsx` - reusable table with search, pagination, and actions column
- List pages pattern: server component (page.tsx) reads searchParams, calls server action, passes data to client component
- Next.js 16 searchParams is a Promise: use `const params = await searchParams` in page components
- "use server" modules: ALL exported functions must be async. Sync helpers must stay private (not exported)
- Client form pattern: client component with useRouter().push() for redirect after successful server action
- Progressive numbering pattern: findFirst with startsWith(prefix) + orderBy desc, parse and increment
- Prisma Decimal creation: `import { Decimal } from "@prisma/client/runtime/library"` then `new Decimal(value)`
- Dynamic line items: array state with updateLine(index, field, value), addLine(), removeLine(index) helpers
- Quote/Invoice form pattern: server component loads clients+taxRates, passes to client form component
- Invoice VAT logic: nazionale=standard(21%), intraUE+VAT=reverse_charge(0%), intraUE-noVAT=standard(21%), extraUE=export_exempt(0%)
- Tax rate type field used for auto-matching: standard, reduced, super_reduced, reverse_charge, export_exempt
- Invoice disclaimer auto-generation: stored on Invoice.disclaimer field, determined by client vatRegime at creation time
- Detail page pattern: server component with `notFound()` for missing records, `export const dynamic = "force-dynamic"`, grid layout for info cards
- "Create from parent" route pattern: /resource/nuova/[parentId] for child entities (e.g., credit note from invoice)
